# 생존편향이 통제된 퇴직연금 ETF 시계열 데이터 수집 계획서

## 0. 개요 및 목적

본 계획서는 XAI 기반 퇴직연금 포트폴리오 진단 모델의 백테스팅 유니버스 구축을 위한 ETF 마스터 데이터셋 수집 절차를 정의합니다.

- **수집 기간:** 2020.01.01 ~ 2025.04.30
- **핵심 원칙:** 생존편향(Survivorship Bias) 제거 — 현재 상장 종목뿐 아니라 수집 기간 내 상장폐지된 종목도 포함
- **최종 목표:** 퇴직연금 적격 ETF의 일별 OHLCV 시계열을 단일 데이터셋으로 구축

## 2. 마스터 유니버스 구축

### 2.1 현재 상장 ETF 수집 (KRX)

**접속 경로:** [KRX 정보데이터시스템](https://data.krx.co.kr) → 통계 → 증권상품 → ETF → 전종목 기본정보

**수집 절차:**
1. 위 경로 접속 후 조회 버튼 클릭
2. 우측 상단 **[다운로드 → CSV]** 버튼으로 저장
3. `raw/krx_etf_current.csv`로 파일명 변경

**수집 항목 (최소 필수):**

| 컬럼명 (KRX 원본) | 용도 |
|---|---|
| 단축코드 | 종목 고유 식별자 (6자리) |
| 한글종목명 | 이름 기반 필터링 |
| 상장일 | 운용 기간 계산 기준 |
| 추적배수 | 레버리지/인버스 판별 |
| 복제방법 | 합성형 파생상품 판별 |
| 기초자산분류 | 자산군 분류 |

### 2.2 상장폐지 ETF 수집 (KIND)

**접속 경로:** [기업공시채널 KIND](https://kind.krx.co.kr) → 상세검색

**검색 조건:**
- 시장구분: `ETF`
- 기간: `2020.01.01 ~ 2025.04.30`
- 보고서명: `상장폐지`

**수집 절차:**
1. 위 조건으로 검색 후 결과 목록 전체 선택
2. **[엑셀 다운로드]** 버튼으로 저장
3. 단축코드, 한글종목명, 상장폐지일 컬럼을 유지하고 `raw/kind_etf_delisted.csv`로 저장

> **확인 사항:** 상장폐지 목록에는 ETF뿐 아니라 ETN도 포함될 수 있음. `종목명` 기준으로 ETN(상장지수채권) 종목은 제거할 것.

### 2.3 병합 (Master Universe 생성)

- KRX 현재 목록 + KIND 폐지 목록을 **단축코드 기준 Outer Join**으로 병합
- 중복 종목(현재 상장이면서 폐지 목록에 오류로 포함된 경우)은 현재 상장 정보를 우선 적용
- `상장폐지일` 컬럼: 현재 상장 종목은 빈 값(NaN)으로 유지
- 결과 저장: `raw/master_universe.csv`

**예상 규모:** 현재 상장 약 900종 + 폐지 약 300~500종 = 총 약 1,200~1,400종

## 3. 퇴직연금 투자 불능 종목 필터링

가격 데이터 수집 전 아래 기준으로 제거하여 수집 비용을 줄입니다.

### 3.1 레버리지 및 인버스 상품 제거

- **기준:** 추적배수가 1이 아닌 종목, 또는 종목명에 아래 키워드가 포함된 종목
- **제거 키워드:** `레버리지`, `인버스`, `2X`, `곱버스`
- **사유:** 퇴직연금 감독규정상 변동성 증폭 파생형 상품 편입 불가

### 3.2 부적격 파생형·합성형 상품 제거

- **기준:** 복제방법이 `합성`이면서 기초자산이 주식·채권이 아닌 종목, 또는 기초자산 분류가 `주식파생형`, `채권파생형`인 종목
- **사유:** 파생상품 위험평가액 비중 40% 초과 펀드 투자 불가

### 3.3 원자재 선물 추종 상품 제거

- **기준:** 종목명에 `선물`이 포함되고 기초자산이 `농산물`, `원유`, `천연가스`, `금속` 등인 종목
- **사유:** 롤오버 비용 발생 및 퇴직연금법상 고위험 원자재 파생상품 투자 제한

> **필터링 후 결과 저장:** `raw/master_filtered.csv`  
> **예상 잔존 규모:** 필터링 후 약 700~900종

## 4. 시계열 가격 데이터 수집

### 4.1 수집 도구

| 도구 | 역할 |
|---|---|
| **pykrx** (메인) | KRX 일별 OHLCV 수집 |
| **증권사 OpenAPI** (보조) | pykrx 트래픽 제한 또는 누락 데이터 발생 시 보완 |

### 4.2 수집 대상 및 항목

- **대상:** `master_filtered.csv`의 전 종목
- **기간:** 2020.01.01 ~ 2025.04.30
- **항목:** 일자, 시가, 고가, 저가, 종가, 거래량

### 4.3 pykrx 수집 시 실무 주의사항

> **반드시 숙지 후 실행할 것**

1. **요청 간격 조절 필수:** 종목당 요청 사이에 최소 `0.5~1초` 지연(sleep)을 넣지 않으면 KRX 서버에서 IP 차단 가능성 있음
2. **예상 소요 시간:** 약 800종 × 요청당 1초 = 최소 **13~15시간** 소요. 야간 실행 권장
3. **진행 상황 저장:** 수집 도중 중단될 수 있으므로 종목별로 수집 완료 여부를 체크하여 중단 시 이어서 재개할 수 있도록 구현할 것 (예: 이미 수집된 종목은 건너뜀)
4. **연도별 분리 수집 고려:** 한 번에 5년치를 요청하면 실패 확률이 높음. 연도별(2020, 2021, …, 2025)로 나눠 수집 후 합치는 방식 권장

### 4.4 수집 로직 (의사코드)

```python
수집_완료_목록 = 이미_수집된_종목_로드()

for 종목코드 in master_filtered['단축코드']:
    if 종목코드 in 수집_완료_목록:
        continue  # 이미 수집된 종목 건너뜀

    try:
        df = pykrx.get_market_ohlcv_by_date('20200101', '20250430', 종목코드)
        df['종목코드'] = 종목코드
        결과_저장(df)
        수집_완료_목록에_추가(종목코드)
    except:
        실패_목록에_기록(종목코드)

    time.sleep(1)
```

## 5. 결측치 처리 및 생존편향 통제

### 5.1 상장 전·폐지 후 구간 — NaN 유지

- 종목의 `상장일` 이전 날짜 및 `상장폐지일` 이후 날짜의 데이터는 **NaN으로 유지**
- 임의값으로 채우지 말 것: 해당 기간에 매매 불가능했음을 모델이 인식해야 함

### 5.2 거래 정지 구간 — Forward Fill

- 상장 유지 기간 중 일시적 거래 정지로 발생한 결측치는 **직전 영업일 종가를 이월(ffill)**
- 단, 연속 결측이 **5영업일 이상** 지속되는 경우 ffill을 적용하지 말고 NaN으로 유지하여 별도 검토

### 5.3 거래량 0 처리

- 거래량이 0인 행은 삭제하지 않고 유지 (공식 거래 정지일일 수 있음)
- 다만 종가가 0인 행은 데이터 오류로 간주하고 NaN 처리

## 7. 수집 완료 후 검증 체크리스트

수집 담당자가 제출 전 반드시 확인해야 할 항목입니다.

- [ ] `master_filtered.csv`에서 레버리지·인버스·선물 키워드 종목이 0건인지 확인
- [ ] `ohlcv_all.parquet`의 `date` 범위가 2020-01-02 ~ 2025-04-30인지 확인
- [ ] 현재 상장 주요 ETF(예: `069500` KODEX200) 데이터가 기간 전체에 걸쳐 존재하는지 확인
- [ ] 상장폐지 종목(예: KIND에서 확인한 코드)의 폐지일 이후 데이터가 NaN인지 확인
- [ ] 종가가 0인 행이 없는지 확인 (`close == 0` 필터링 후 행 수 = 0)
- [ ] 실패 목록(`failed_tickers.csv`)이 있다면 내용을 공유하여 보완 여부 협의
- [ ] 파일 크기 및 총 행 수 위 표 예상치와 크게 벗어나지 않는지 확인
